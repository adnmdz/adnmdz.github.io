Project Entry

QIS Capsule Tracking System

For my capstone project with ARTMS, I worked on a system to track radiopharmaceutical capsules inside shielded pneumatic hoses. The goal was simple: give operators clear position feedback without opening the system or interrupting production.

Overview

Project overview

What Track capsule position inside shielded transfer hoses.
Why Reduce downtime and operator exposure.
How Compare concepts, prototype sensors, and test them.
Outcome Live feedback was demonstrated in controlled testing.

Problem

ARTMS moves radioactive capsules through sealed hoses during isotope production. If a capsule jams, operators cannot directly see where it is.

Our solution was an external sensing system that gives live capsule position feedback without opening the process or disturbing the sealed hose.

Constraints

The design had to meet ISO 7 cleanliness, ALARA safety, tight space, sealed transport, and existing workflow constraints.

We chose externally mounted, non-invasive sensing because it preserved the filtered hose, reduced contamination risk, and was easier to integrate into the client workflow.

Design Story

How the solution evolved

The design moved from broad concepts to a testable sensing system.

Define the problem

We defined the failure case: jammed capsules were hard to locate and current recovery methods were disruptive.

Map the constraints

Any solution had to preserve the sealed hose, fit tight geometry, and avoid adding new safety or maintenance risks.

Generate competing concepts

We compared robotic approaches with sensing options before committing.

Use a weighted decision process

Sensor concepts scored best because they were simpler, safer, and easier to integrate.

Prototype and validate

We built a bench test rig, captured sensor data, and confirmed that live feedback was feasible.

Key Decision

The final design used multiple sensing modes.

Accelerometer sensing scored highest, but the final system used redundancy. Inductive, accelerometer, and time-of-flight sensing covered different weaknesses and reduced the risk of misleading readings.

Final System

What we built

System build

We built a bench-scale prototype that combined mechanical coupling, sensing hardware, an interface PCB, and an HMI to validate live capsule tracking.

  • Distributed sensing nodes mounted along the hose.
  • A central interface PCB to collect sensor data.
  • An HMI and PLC-facing interface for live feedback.
  • A test bench for repeatable prototype validation.
  • My work focused on the embedded and experimental side, especially firmware, sensing, and test validation.

Sensor Stack

Why three sensing modes were stronger than one

Inductive

LDC1101 path

A PCB coil changes resonant behavior when the aluminum capsule passes nearby. It works through the hose wall and suits a sealed system.

Accelerometer

BMA400 path

Capsule motion creates vibrations in the hose and coupler. It was highly sensitive but required care around environmental noise.

Time of flight

VL53L0X path

A transparent coupler allowed direct distance sensing. It offered a clear signal but required a more deliberate mechanical interface.

Testing

How we checked whether it actually worked

Sensor data plot

Figure 13 shows accelerometer and time-of-flight data collected during testing. The spikes align with capsule motion and indicate when the system sensed the capsule passing through the setup.

The key result was feasibility: the prototype produced detectable sensor responses and usable live feedback in controlled testing.
Limitations

Testing limits

The main limitations came from prototype maturity and test conditions. We were unable to fully test the inductive sensor path, so that part of the system still needs additional validation.

We also had issues with the clarity of the 3D-printed coupling, which affected the optical path and limited time-of-flight performance. More broadly, the test setup still faced air leakage, limited cycle count, and no live validation in the full QIS infrastructure.

FMEA showed the highest-risk issues were coupler integrity and sensor reliability, so mechanical sealing and stable sensing remained the main design priorities.
What Comes Next

Future improvements

Production-ready electronics

Refine the sensor PCB with a 3-4 layer stack-up, simpler connectors, and fewer prototype-only parts.

Stronger industrial integration

Fabricate the interface PCB and replace USB testing links with plant-ready protocols such as MODBUS or PROFINET.

Sensor reliability

Add startup tests, periodic calibration, and improved ToF packaging or coupler interfaces to improve detection reliability.
